# 작업 수행 계획서

## 작업 개요
- 작업명: BASE_PATH 수정 및 시스템 상태 로깅 보강 작업 계획
- 작업 유형: 신규 작업
- 작성 일시: 2026-05-20

## 배경
개발 작업 리포트 허브 메인 화면에서 시스템 상태, 보고서 바로가기, 보고서 목록이 모두 로딩 실패 상태로 표시되고 있다. 현재 증상은 메인 HTML 렌더링 이후 초기 데이터 로딩 단계에서 실패가 발생했음을 시사한다.

추가로, 시스템 상태 문제를 다시 확인할 수 있도록 운영 로그를 남길 수 있는 구조가 필요하다. 기존에는 Supabase 설정이 placeholder 상태여서 브라우저 초기화 실패를 지속적으로 추적하기 어려웠다.

## 목표
1. BASE_PATH 계산 오류 가능성을 실제 수정 대상으로 정의한다.
2. 시스템 상태 확인을 위한 로그 수집 구조를 Supabase 기준으로 설계한다.
3. 수정 전 근거와 수정 후 검증 기준을 명확히 남긴다.
4. 이번 작업을 신규 보고서 세트로 분리해 기록한다.

## 주요 원인 가설
- GitHub Pages 루트 배포인데도 하위 프로젝트 경로 기준으로 BASE_PATH가 계산될 수 있다.
- 그 결과 `data/reports.json` 요청 경로가 잘못 계산되어 메인 초기화가 실패한다.
- 실패 원인이 발생해도 현재는 브라우저 콘솔 외에는 운영 로그가 충분히 남지 않는다.

- Supabase 로그 테이블 도입 계획 정리

- 검증 시나리오 수립
